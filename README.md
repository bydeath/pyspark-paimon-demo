# **Apache Paimon T+1 增量数据处理流水线 (Spark 实现)**

本项目提供了一个基于 Apache Paimon 和 Apache Spark 的 T+1 离线增量数据处理参考实现。该流水线旨在解决大数据场景下维度变更触发的存量数据回填问题，通过数据湖特有的增量能力实现高效的宽表构建。

## **1\. 核心技术栈**

* **Apache Paimon (1.3.1)**：提供数据湖存储、LSM-Tree 索引及快照管理。  
* **Apache Spark (3.5.x)**：负责分布式批处理计算及 ETL 逻辑。  
* **Python (3.12.3)**：流水线调度与任务控制。

## **2\. 方案架构设计**

### **2.1 存储策略**

* **Changelog-Producer: none**：在 ODS 层配置 none 模式。此模式下不产生额外的 Changelog 文件，避免了“双写惩罚”，从而在 T+1 批量写入时获得更高的吞吐量。  
* **Merge-Engine: partial-update**：DWD 宽表采用局部更新引擎。事实增量流与维度增量流可独立更新宽表中的特定列，引擎根据主键自动进行物理层面的合并。  
* **复合主键映射表**：关系索引表采用 (维度外键, 事实主键) 作为复合主键。相比数组结构，该设计利用 LSM-Tree 的主键特性实现天然去重，并规避了分布式环境下的乱序覆盖风险。

### **2.2 游标管理 (基于 Paimon Tag)**

本项目放弃了传统的外部文件或第三方数据库记录偏移量的方式，完全采用 Paimon 原生的 **Tag** 机制进行水位线管理：

* **无状态游标**：通过查询 Paimon 系统表 $tags 获取上次成功执行的位点。  
* **数据生命周期锁定**：被 Tag 引用的快照及其关联的数据文件将受到保护，不会被 Paimon 的过期清理机制回收，确保了增量链路的稳定性。  
* **自动回收机制**：内置 cleanup\_old\_tags 函数，根据预设的保留阈值自动清理过期 Tag，释放存储空间。

### **2.3 数据处理流程**

1. **事实驱动同步**：捕获 ODS 事实表增量，关联维度表最新状态写入 DWD，并维护关系映射表。  
2. **维度驱动回填**：捕获 ODS 维度表增量，通过映射表定位受影响的事实记录，仅回填 DWD 中的维度属性列。  
3. **物理对齐优化**：任务末尾调用 Paimon 系统过程 sys.compact 对 DWD 宽表进行 Full Compaction，提升下游 OLAP 查询性能。

## **3\. 快速开始**

### **预备条件**

* 已部署 Spark 3.5.x 集群或本地环境。  
* 已下载对应版本的 Paimon Spark Connector JAR 包。

### **配置与运行**

1. 修改脚本中的 paimon\_spark\_jar 变量，指向您的 JAR 包物理路径。  
2. 配置 warehouse\_path，在生产环境下建议指向 HDFS 或 S3 路径。  
3. 执行脚本：  
   python paimon\_incremental\_pipeline.py

## **4\. 仓库结构**

* paimon\_incremental\_pipeline.py: 核心批处理逻辑脚本。  
* paimon\_warehouse/: (自动生成) Paimon 元数据与数据文件存储目录。

## **5\. 免责声明与注意事项**

* 本代码仅作为技术参考示例，在投入生产环境前，请根据具体业务量级调整 bucket 数量及并行度。  
* 当 changelog-producer 设置为 none 时，增量读取可能会包含部分因 Compaction 产生的重复数据，需依赖下游表的主键幂等性进行吸收。
